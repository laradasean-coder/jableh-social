# النشر من جديد على نظافة — دليل سريع

> آخر نسخة على GitHub كانت 16. هذه النسخة v17 تُنشر نظيفة بالكامل.

## 1) الكود (GitHub + Vercel)
- استبدل محتوى المستودع بمحتوى مجلد `jabla-v17` بالكامل، ثم ادفع إلى الفرع الرئيسي.
- Vercel سيبني تلقائياً. تأكد من متغيّرات البيئة في Vercel:
  - `VITE_SUPABASE_URL`
  - `VITE_SUPABASE_ANON_KEY`
  - (اختياري) `VITE_CLOUDINARY_CLOUD_NAME`, `VITE_CLOUDINARY_PRESET`

## 2) قاعدة البيانات — اختر حالتك

### أ) مشروع Supabase **جديد/فارغ** (الأنظف)
نفّذ ملفاً واحداً فقط في SQL Editor:
```
supabase/schema_full.sql
```
يُنشئ كل الجداول والسياسات والدوال والمُحفّزات والبذور (الوحدات/المناطق) دفعة واحدة، وكل الإصلاحات مدمجة. (آمن لإعادة التشغيل.)

### ب) قاعدة فيها **بيانات تريد الاحتفاظ بها**
لا تستخدم `schema_full.sql` (سيُعيد البذور فقط، لكن لا تخاطر). طبّق بالترتيب:
```
008 (المصحّح) ← 014 ← 015 ← 016 ← 017
```
كلها idempotent.

## 3) التخزين (Storage)
أنشئ bucket باسم `uploads` (Public).

## 4) الدوال (Edge Functions)
```
supabase secrets set GROQ_API_KEY=gsk_xxxxx
supabase functions deploy ai-assistant
supabase functions deploy admin-users
```
(لا أسرار إضافية لـ admin-users — مفاتيح Supabase مُحقَنة تلقائياً.)

## 5) إنشاء أول مدير (مهم)
بعد النشر، التسجيل الذاتي يمنح دور `association` فقط (إجراء أمني). لإنشاء أول مدير:
1. أنشئ مستخدماً من Supabase → Authentication → Add user (بالبريد وكلمة المرور).
2. نفّذ في SQL Editor (ضع بريد المدير):
```sql
UPDATE public.profiles SET role = 'admin'
WHERE id = (SELECT id FROM auth.users WHERE email = 'admin@yourdomain.com');
```
بعدها يدير بقية الموظفين والأدوار من صفحة «الموظفون» داخل التطبيق.

> ملاحظة: باب `admin/admin` التطويري للمعاينة فقط (لا ينشئ مستخدمين حقيقيين).

## التحقق الآلي
`.github/workflows/ci.yml` يطبّق الهجرات + البذور + تأكيدات قاعدة البيانات ويبني الواجهة في كل دفع — يضمن عدم تكرار الأخطاء.
