# دليل التحديث — الإصدار v17

ملخص التعديلات المنفذة (البنود 1–7) وخطوات التفعيل.

## ما تم تنفيذه

1. **الهوية البصرية (شعار النسر)**
   - شعار جديد دقيق (SVG متجه) مستخرج من الصورة: `src/components/ui/SyriaEmblem.jsx`.
   - أصول: `public/emblem.svg`, `public/favicon.svg`, `public/icon-192.png`, `public/icon-512.png`, `public/apple-touch-icon.png`.
   - تحديث `index.html`, `public/manifest.json`, `vite.config.js` (لون الثيم `#0D4A35`).

2. **صفحة المدير قابلة للتعديل**
   - `AdminPage.jsx` يقرأ بياناته من `site_settings` (الاسم، المسمى، النبذة، السيرة، الرؤية، البريد، الهاتف، الصورة).
   - التحرير من: **إدارة المحتوى → مجموعة «صفحة المدير»**. عند تغيّر المدير يُعدّله المدير بلا كود.

3. **ملفات/سجلات الوحدات (Excel / Word / صور)**
   - داخل صفحة التنمية الريفية، بعد تسجيل دخول الوحدة: تبويب **«سجلّات وملفات الوحدة»** للرفع والعرض والحذف.
   - يُخزَّن في Supabase Storage (bucket `uploads`) وبيانات الملف في جدول `uploaded_files`.

4. **التقارير اليومية/الأسبوعية**
   - داخل الوحدة: تبويب **«رفع تقرير للمدير»** (نوع/تاريخ/عنوان/محتوى).
   - تُرسَل إلى جدول `unit_reports` وتظهر للمدير في صفحة **«تقارير الوحدات»** (`/unit-reports`) مع إشعار تلقائي.

5. **المساعد الذكي Groq**
   - `supabase/functions/ai-assistant/index.ts` يستخدم Groq (واجهة OpenAI متوافقة).

6. **تسجيل الجمعيات + اعتماد المدير**
   - في صفحة الدخول: **«سجّل جمعيتك»** (إنشاء حساب).
   - يُنشأ الحساب بدور `association` وحالة `pending` (لا يستطيع الدخول حتى الاعتماد).
   - المدير يعتمد/يرفض من صفحة **الجمعيات** (لوحة «طلبات بانتظار المراجعة»).

7. **الدور كقائمة منسدلة (SQL)**
   - تحويل `profiles.role` إلى نوع `ENUM user_role` ليظهر كقائمة منسدلة في محرّر Supabase.
   - (القائمة المنسدلة موجودة أصلاً في صفحة «الموظفون» داخل التطبيق).

## ⚠️ إصلاح أمني مهم
كان مُحفِّز `handle_new_user` يثق بدور المستخدم القادم من العميل (`raw_user_meta_data->>'role'`) — ثغرة تصعيد صلاحيات تتيح طلب دور `admin` عند التسجيل الذاتي. تم تعديله ليفرض دائماً دور `association`؛ المدير يرفع الأدوار يدوياً من صفحة الموظفين.
> ملاحظة: باب الدخول `admin/admin` التطويري بقي كما هو بناءً على طلبك.

## خطوات التفعيل بعد الرفع

1. **قاعدة البيانات** — نفّذ المigration الجديد في Supabase (SQL Editor أو CLI):
   ```
   supabase/migrations/014_reports_signup_roles.sql
   ```
2. **Storage** — تأكد من وجود bucket باسم `uploads` (Public) وسياساته تسمح للموظفين/المدير/رؤساء الوحدات بالرفع.
3. **مفتاح Groq** (تضيفه أنت):
   ```
   supabase secrets set GROQ_API_KEY=gsk_xxxxx
   supabase functions deploy ai-assistant
   ```
4. **النشر** — ادفع إلى Git ليعيد Vercel البناء، أو `npm run build`.

الـ 14 ملف هي **هجرات متسلسلة** (لا تتعارض) وكلها idempotent. لا حاجة لدمجها على قاعدة قائمة.

## إصلاحات اكتُشفت أثناء المحاكاة التشغيلية (مهمة)
- **015 (حرج):** كانت سياسات RLS على `profiles` تستعلم عن `profiles` نفسه ⇒ "infinite recursion" يُعطّل أي استعلام لمستخدم حقيقي. أُصلح عبر دالة `current_user_role()` بنمط `SECURITY DEFINER` وإعادة بناء السياسات. **يجب تنفيذ `015_fix_profiles_rls_recursion.sql`.**
- **008:** كان `LIMIT` يسبق `UNION ALL` بلا أقواس ⇒ دالة `global_search` لم تكن تُنشأ. أُصلح بإضافة الأقواس.

### ترتيب التنفيذ على قاعدة قائمة
نفّذ بالترتيب: `008` (إعادة) ← `014` ← `015`. (كلها idempotent وآمنة لإعادة التشغيل.)

## تحديثات الدفعة الثانية (البنود 2–7 + سجل الدوام)
- **الهوية:** أُعيد توليد الشعار من الصورة النظيفة (متجه دقيق، ذهبي #B7A57A) — `SyriaEmblem` + الأيقونات.
- **016:** تشديد قراءة الجمعيات (الجمهور يرى المعتمدة فقط) + توحيد كل فحوص الأدوار عبر `current_user_role()` (أداء أوضح، لا تكرار).
- **017:** جدول `attendance` (سجل دوام فعلي) + دالة `send_attendance_sheet` (إرسال الكشف لموظف مسؤول يظهر في إشعاراته/بريده) + `list_staff_recipients`.
- **سجل الدوام (الواجهة):** تبويب «سجل الدوام» داخل الوحدة — قائمة الموظفين، تحديد الحالة (حاضر/غائب/إجازة/متأخر/مهمة)، حفظ، وإرسال الكشف لموظف مسؤول مختار.
- **تقارير الوحدات:** تصدير PDF/Excel في صفحة المدير، والتقرير مرتبط بالمُرسِل عبر `auth.uid`.
- **تمرير باللمس:** سحب يمين/يسار للتنقل بين الأقسام على الجوال.
- **ضغط الصور:** تصغير وإعادة ترميز الصور قبل الرفع تلقائياً.
- **CI:** `.github/workflows/ci.yml` يشغّل الهجرات + بذور + تأكيدات قاعدة البيانات، ويبني الواجهة ويشغّل الاختبارات في كل دفع.
