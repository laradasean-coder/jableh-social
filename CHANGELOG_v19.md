# تحديث جبلة v19 — إصلاح الشاشة البيضاء عند دخول الوحدة

## التشخيص
بعد فحص دقيق:
- صفحة الوحدات **لا تحتوي انهياراً ثابتاً** في الكود بعد إصلاح v18
  (كل المكوّنات مُعرّفة، و`UnitFilesReports` يعمل بمفرده دون مشاكل).
- أرجح سبب لشاشة بيضاء «كأنها كراش» **تتجاوز شاشة الخطأ**: فشل تحميل
  ملف شيفرة الصفحة (lazy chunk) لأن الـ Service Worker (PWA) ما زال
  يشير لملفات النسخة القديمة بعد إعادة النشر — فيظهر فراغ عند فتح صفحة
  لم تُحمّل شيفرتها بعد (مثل صفحة الوحدات).

## الإصلاحات في v19
1. **تحميل ذكي للصفحات** `src/lib/lazyWithRetry.js`:
   عند فشل تحميل أي صفحة، يُعاد تحميل التطبيق تلقائياً مرة واحدة لجلب
   أحدث الملفات بدل ترك المستخدم أمام شاشة بيضاء. طُبّق على كل المسارات في `App.jsx`.
2. **تحصين الـ PWA** `vite.config.js`:
   - `skipWaiting` + `clientsClaim`: النسخة الجديدة تتولّى فوراً.
   - `cleanupOutdatedCaches`: حذف ملفات النسخ القديمة.
   - `navigateFallback`: ضمان عمل تنقّل التطبيق أحادي الصفحة.
3. **حدود خطأ محلية** حول `UnitFilesReports` داخل صفحة الوحدات:
   أي خلل في بياناتها يظهر كرسالة صغيرة بدل التأثير على الصفحة.
4. تأكيد إصلاح v18: استيراد `UnitFilesReports` (كان غيابه يُعطّل توسعة الوحدة).

## مهم جداً عند النشر (لإزالة الشاشة البيضاء نهائياً)
1. ابنِ النسخة: `npm install && npm run build`.
2. انشر محتوى مجلد `dist`.
3. **امسح كاش المتصفح / أزل تثبيت PWA القديم مرة واحدة** على الأجهزة التي
   ظهرت فيها المشكلة (أو انتظر تحديث الـ Service Worker تلقائياً عند أول
   إعادة فتح). بعد v19 سيتولّى التطبيق ذلك ذاتياً مستقبلاً.

## للاختبار السريع
- ادخل كمدير → افتح «التنمية الريفية» → وسّع «وحدة بيت ياشوط» → يجب أن
  تظهر التفاصيل (الخدمات/المشاريع/الملفات/الدوام/التقارير) دون شاشة بيضاء.
- ادخل بحساب رئيس وحدة → يُفترض أن ترى وحدته فقط.
