# دليل الإصدار 13 — المنصة المتكاملة

## ما الجديد في هذا الإصدار

### مقترحاتك المنفّذة
1. **إصلاح سياسات profiles** (migration 009) — أي مستخدم تضيفه من Authenticator يحصل على profile تلقائياً عبر trigger، بلا أي قيود.
2. **زر تثبيت PWA** — يظهر تلقائياً على Android والكمبيوتر، مع دليل تثبيت لـ iPhone.

### التطويرات المميزة
3. **لوحة Kanban للإغاثة** — اسحب الطلبات بين المراحل، مع تحديث فوري.
4. **تقرير وزاري PDF رسمي** — في صفحة التقارير، اختر الشهر والسنة واضغط "إصدار التقرير الوزاري".
5. **نظام SLA** — كل طلب له مهلة، المتأخر يظهر بشارة حمراء في الكانبان.
6. **إشعار المتقدم** — عند الموافقة/الرفض يصله SMS تلقائياً (عبر مزوّد SMS المُعدّ).
7. **سجل الأمان** — صفحة `/security` تعرض آخر عمليات الدخول والأجهزة.
8. **النسخ الاحتياطي** — تصدير كل البيانات JSON أو Excel بضغطة.
9. **صفحة عن الدائرة** — `/about` صفحة تعريفية عامة بالرؤية والرسالة والأرقام.
10. **مركز الإدارة المُوسّع** — كل الأقسام من صفحة `/content`.

---

## خطوات النشر (مهم جداً — بالترتيب)

### 1. شغّل migrations الجديدة في Supabase SQL Editor
```
009_profiles_autocreate_fix.sql   ← يصلح إضافة المستخدمين
010_sla_and_security.sql          ← SLA + سجل الدخول
```
> ملاحظة: إن لم تكن شغّلت 007 و008 سابقاً، شغّلها أولاً.

### 2. أنشئ Storage bucket (إن لم يكن موجوداً)
Supabase Dashboard > Storage > New Bucket
- الاسم: `uploads`
- Public: نعم

### 3. فعّل Realtime
Database > Replication > فعّل:
`beneficiaries, relief_requests, notifications, support_messages, support_threads`

### 4. ارفع على Vercel
```bash
npm install
npm run build
# ثم ارفع، أو اربط GitHub بـ Vercel
```

### 5. أضف موظفاً جديداً (الآن يعمل بلا مشاكل)
- Supabase > Authentication > Add User
- سيُنشأ له profile تلقائياً بدور "staff"
- لترقيته لمدير: عدّل الدور من صفحة الموظفين أو من جدول profiles

## مزوّد SMS (لإشعار المتقدمين)
بما أن Twilio محظور، استخدم أحد البدائل:
```bash
supabase secrets set SMS_PROVIDER=unifonic
supabase secrets set UNIFONIC_APP_SID=xxx
supabase functions deploy send-sms
```
الإشعار الداخلي يعمل دائماً حتى بدون SMS.
