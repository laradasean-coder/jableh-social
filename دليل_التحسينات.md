# دليل تحسينات الأداء والتخزين

## 1. استضافة صور الحالات على Cloudinary (موفّر للمساحة)

### الإعداد (5 دقائق، مرة واحدة)
1. أنشئ حساباً مجانياً: cloudinary.com (25 غيغابايت مجاناً)
2. من Dashboard انسخ **Cloud Name**
3. Settings ⚙️ > Upload > Upload presets > Add upload preset:
   - **Signing Mode**: Unsigned
   - **Folder**: jabla-relief-cases
   - احفظ واسم الـ preset
4. أضف القيم في Vercel (Environment Variables) أو .env:
   ```
   VITE_CLOUDINARY_CLOUD_NAME=اسم_الكلاود
   VITE_CLOUDINARY_PRESET=اسم_البريست
   ```

### كيف يعمل
- صورة الحالة تُضغط في المتصفح (1200px، جودة 75%) ثم تُرفع لـ Cloudinary.
- الرابط فقط يُحفظ في Supabase (نص قصير بدل ملف ثقيل).
- إن لم تُضبط Cloudinary، يتراجع تلقائياً لـ Supabase Storage (لا يتعطل شيء).

> صور الوحدات والأخبار تبقى على Supabase (عامة وقليلة).
> صور الحالات الحساسة والكثيرة تذهب لـ Cloudinary.

## 2. منع تجميد Supabase (GitHub Actions)

الملف `.github/workflows/keep-alive.yml` يرسل نبضة كل 3 أيام.

### الإعداد
1. ارفع المشروع على GitHub
2. Settings > Secrets and variables > Actions > New secret:
   - `SUPABASE_URL` = رابط مشروعك
   - `SUPABASE_ANON_KEY` = المفتاح العام
3. تلقائياً سيعمل كل 3 أيام — لن يتجمّد المشروع أبداً.

---

## 3. تنظيف تلقائي لقاعدة البيانات

Migration `013_cleanup_storage.sql` يضيف دالة `cleanup_old_logs()`:
- تحذف سجل العمليات الأقدم من 6 أشهر
- تحذف الإشعارات المقروءة والدخول الأقدم من 3 أشهر

شغّلها يدوياً من حين لآخر:
```sql
SELECT public.cleanup_old_logs();
```
أو في خطة Pro، جدولها شهرياً عبر pg_cron (معلّق في الملف).

## الأثر المتوقع
| التحسين | التوفير |
|---------|---------|
| Cloudinary للصور | يفرّغ ~90% من ضغط التخزين |
| ضغط الصور | كل صورة أصغر 70-80% |
| keep-alive | صفر تجميد |
| تنظيف السجلات | يمنع تضخّم قاعدة البيانات |

بهذه التحسينات، الخطة المجانية تكفيك فترة أطول بكثير قبل الحاجة لـ Pro.
